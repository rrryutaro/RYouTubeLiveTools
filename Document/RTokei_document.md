# RTokei — デスクトップ時計アプリ ドキュメント

作成日: 2026/03/28  
バージョン: 1.0

---

## 概要

Windows用のデスクトップ常駐型デジタル時計アプリです。  
C# 製の既存時計アプリ（常に最前面機能に不具合あり）をベースに、Python/tkinter で新規作成しました。

### 表示レイアウト

```
┌─────────────────────────┐
│   2026/03/28（土）       │  ← 日付 + 曜日
│  ─────────────────────  │  ← 区切り線
│       12:00:00          │  ← 時刻（HH:MM:SS）
└─────────────────────────┘
```

デフォルト配色は深夜ネイビー系（背景: 濃紺 / 時刻: アイスホワイト / 日付: くすんだブルー）。

---

## ファイル構成

| ファイル名 | 役割 |
|---|---|
| `clock.py` | アプリ本体（Python スクリプト） |
| `build_exe.bat` | EXE ビルド用バッチファイル |
| `clock_settings.json` | 設定保存ファイル（初回起動時に自動生成） |
| `RTokei.exe` | ビルド後の実行ファイル（`dist\` フォルダに生成） |

> **注意:** `clock_settings.json` を削除するとすべての設定がデフォルトにリセットされます。

---

## EXE のビルド方法

### 必要環境

- Windows 10 / 11
- Python 3.9 以上（3.11 推奨）
- pip（Python に同梱）

### 手順

1. `clock.py` と `build_exe.bat` を**同じフォルダ**に置く
2. `build_exe.bat` をダブルクリック
3. PyInstaller が未インストールの場合は自動でインストールされる
4. 完了後、`dist\RTokei.exe` が生成される

```
[フォルダ構成例]
C:\MyApps\RTokei\
  ├─ clock.py
  └─ build_exe.bat

↓ ビルド後

C:\MyApps\RTokei\
  ├─ clock.py
  ├─ build_exe.bat
  ├─ clock_settings.json   ← 起動後に自動生成
  └─ dist\
       └─ RTokei.exe       ← これを使う
```

> **重要:** `build_exe.bat` は **Shift-JIS (CP932)** で保存されています。  
> テキストエディタで編集・保存する場合は必ず CP932 を選択してください。  
> UTF-8 で保存するとコマンドが文字化けして動作しなくなります。

## 使い方

### 基本操作

| 操作 | 方法 |
|---|---|
| ウィンドウ移動 | アプリ本体をドラッグ |
| 設定を開く | 右クリック → 設定を開く |
| サイズ変更（クイック） | 右クリック → サイズ → プリセット名を選択 |
| 常に最前面 切替 | 右クリック → 📌 常に最前面 |
| 背景透過 切替 | 右クリック → 💧 背景透過 |
| タイトルバー 切替 | 右クリック → 🔲 タイトルバー |
| 終了 | 右クリック → ✖ 終了 |

> タイトルバーを非表示にしていても、アプリ本体をドラッグして移動できます。

### モニター端への吸着（スナップ）

ウィンドウをモニターの端から **20px 以内** に近づけてマウスボタンを離すと、自動的にその端に吸着します。

- 上下左右の 4 辺すべてに対応
- 角（コーナー）にも吸着可能（上端 + 左端 など同時適用）
- **マルチモニター対応**（2枚目・3枚目のモニターの端にも正しく吸着）
- タスクバーの位置を自動認識し、タスクバーに重ならない位置にスナップ

## 設定画面の詳細

右クリック → 「設定を開く」で設定ウィンドウが開きます。

### フォント

| 項目 | 内容 |
|---|---|
| フォント名 | Windows にインストール済みのフォント名を直接入力（例: `メイリオ`、`MS ゴシック`、`游ゴシック`） |
| フォントサイズ | 日付と時刻を個別に指定（8〜200pt） |
| スタイル | 太字・斜体を個別に ON/OFF |

### カラー

| 項目 | 内容 |
|---|---|
| 日付の色 | 1段目（日付・曜日）の文字色 |
| 時刻の色 | 2段目（HH:MM:SS）の文字色 |
| 背景色 | ウィンドウ全体の背景色 |

色はボタンをクリックすると Windows 標準のカラーピッカーが開きます。

### サイズプリセット

| プリセット | ウィンドウサイズ | 日付フォント | 時刻フォント |
|---|---|---|---|
| 極小 | 220 × 70 px | 13 pt | 24 pt |
| 小（デフォルト） | 290 × 90 px | 16 pt | 32 pt |
| 中 | 360 × 112 px | 20 pt | 40 pt |
| 大 | 480 × 148 px | 26 pt | 54 pt |
| 極大 | 640 × 196 px | 34 pt | 72 pt |

> プリセット選択後もフォントサイズ欄を手動変更すれば、自由なサイズにカスタマイズできます。

### ウィンドウオプション

| 項目 | 内容 |
|---|---|
| 常に最前面 | ON にすると他のウィンドウの前面に常に表示される |
| 背景透過 | ON にすると背景色が透明になる（文字だけ浮かぶ表示） |
| タイトルバー | OFF にするとウィジェット風の枠なし表示になる |

## 設定の自動保存

以下のタイミングで `clock_settings.json` に自動保存されます。

- 設定画面で「適用」をクリックしたとき
- 右クリックメニューでトグルを切り替えたとき
- **アプリを終了したとき（ウィンドウの位置・サイズも保存）**

次回起動時は保存された位置・設定で自動的に復元されます。

## 技術情報（今後の修正・拡張時の参考）

### 構成クラス

| クラス名 | 役割 |
|---|---|
| `ClockApp` | メインウィンドウ、時刻更新、ドラッグ・スナップ処理 |
| `SettingsWindow` | 設定ダイアログ（tk.Toplevel） |

### 主要メソッド（ClockApp）

| メソッド | 内容 |
|---|---|
| `_tick()` | 毎秒時刻を更新。`microsecond` を使ったドリフト補正あり |
| `_on_drag_end()` | マウスリリース時にスナップ判定・吸着処理 |
| `_apply_settings()` | 設定を UI 全体に反映し JSON 保存 |
| `_relayout()` | ウィンドウサイズに合わせてラベル・区切り線を再配置 |
| `_toggle_topmost()` | 最前面切替。`root.lift()` + 50ms 後再適用で確実化 |
| `_toggle_titlebar()` | タイトルバー切替。`overrideredirect` 変更後 100ms で topmost 再適用 |

### スナップ処理の仕組み

Windows API を使ってウィンドウが乗っているモニターを特定し、そのモニターの作業領域（`rcWork`）を基準にスナップします。

```
MonitorFromPoint()  → 対象モニターのハンドルを取得
GetMonitorInfoW()   → rcWork（タスクバー除いた作業領域）を取得
吸着距離 20px 以内なら geometry を作業領域端に移動
```

Windows API が使えない環境では `winfo_screenwidth/height` にフォールバック。

### バッチファイルの注意点

`build_exe.bat` は **Shift-JIS (CP932) + CRLF** で保存することが必須です。  
UTF-8 で保存すると Windows のコマンドプロンプトが文字化けし、コマンドが壊れて動作しなくなります（過去に発生した不具合）。

---

## 既知の制限・注意事項

- **Windows 専用**です（tkinter の `-transparentcolor`、Windows API 依存のため）
- 背景透過とタイトルバー非表示を同時に使うと、稀に最前面の再適用が必要になる場合があります（右クリック → 常に最前面 を ON/OFF して再設定してください）
- フォント名に存在しないフォントを入力した場合、システムデフォルトフォントにフォールバックします
