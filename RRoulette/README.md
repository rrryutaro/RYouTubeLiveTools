# RRoulette

バージョン: v0.2.0

YouTubeライブ配信向けルーレットツール。OBSのウィンドウキャプチャに対応しています。

---

## ダウンロードして使う（一般ユーザー向け）

[Releases](../../releases) ページから最新の `RRoulette.exe` をダウンロードして実行してください。インストール不要です。

> **Windows SmartScreen 警告について**
>
> 初回実行時に「Windows によって PC が保護されました」という警告が表示される場合があります。
> これはアプリの不具合ではなく、コード署名証明書の有無・配布実績（reputation）に関する Windows の仕組みです。
>
> 警告が出た場合は「詳細情報」→「実行」で起動できます。
> ソースコードはこのリポジトリで公開しています。不安な場合はソースから直接実行することもできます（[ソースから実行する](#ソースから実行する開発者向け) を参照）。

## 使い方 / 操作方法

### 基本操作

| 操作 | 動作 |
|------|------|
| 画面クリック または スペースキー | スピン開始 / 停止 |
| スピン中に2連打（400ms以内） | ダブル停止（設定秒数で減速） |
| スピン中に3連打 | トリプル停止（即時停止も可） |
| 右クリック | メニューを表示 |

### コントロールボックス

メインパネル右上に常時表示されるボタン群です。

| ボタン | 動作 |
|--------|------|
| ≡ | 項目リスト 表示/非表示 |
| ⚙ | 設定パネル 表示/非表示 |
| ━ | 最小化 |
| □ / ❐ | 最大化 / 元に戻す |
| ✕ | 閉じる |

### 右クリックメニュー

補助的な簡易操作メニューです。

- **表示**: コントロールボックス・リサイズグリップ・ログのON/OFF
- **サイズ**: S / M / L のプリセットサイズに変更
- **終了**

### 項目リスト

- 右側のサイドバーで項目を管理します
- **編集ボタン** → テキスト欄で項目を編集（1行1項目） → **保存ボタン**
- コンボボックスでグループを切り替え・追加・削除できます
- 各項目の左にあるチェックボックスで ON/OFF を切り替えられます（OFF の項目はルーレットから除外されます）
- 各項目行の「確率」欄で確率設定（デフォルト / 固定% / 重み）を選択できます
- 各項目行の「分割」欄でルーレット上の区画数（1〜10）を設定できます
- タイトル行の **独立表示** ボタンで項目リストを別ウィンドウに切り出せます。**メインに戻す** で元に戻ります

### マーカー

- ゴールドの三角マーカーをドラッグして任意の位置に移動できます
- 設定パネルのプリセット（上/右/下/左）でも変更できます

### ログ表示

- スピン結果が画面左端にリアルタイムで表示されます（新しいものが上）
- 結果にカーソルを合わせると、そのときのグループ・項目構成がポップアップ表示されます
- 右クリックメニューの「ログを表示」でON/OFFを切り替えられます

### 設定パネル

コントロールボックスの **⚙** ボタンから開きます。

- **ウィンドウ表示**: 最前面表示・背景透過のON/OFF
- **ルーレット文字表示**: 表示方向（横向き・縦向きなど5種）・文字サイズモード（省略/収める/縮小）・ドーナツ穴サイズ
- **ポインター位置**: マーカー基準位置のプリセット選択
- **ログ設定**: 日時記録・四角囲み・前面表示のON/OFF、ログ出力・ログ削除
- **音量**: スピン音・決定音の音量
- **スピン音 / 決定音**: パターン選択・カスタムファイル指定
- **操作設定**: スピン時間（通常 / ダブル停止 / トリプル停止）
- **配置設定**: 項目配置順方向・ルーレット回転方向

設定パネルのタイトル行にある **独立表示** ボタンで、設定パネルを別ウィンドウとして切り出せます。**メインに戻す** で元に戻ります。

### ログ出力

設定パネルの「ログ設定」グループからログをファイルに保存できます:

- **ログ出力（結果のみ）**: シンプルなテキスト形式（.txt）
- **ログ出力（グループ・項目付き）**: グループ名と項目構成を含む詳細形式（.json）
- **ログ削除**: 現在のログを全件削除

## トラブルシューティング

**ウィンドウが画面外に出てしまった**
→ EXE と同じフォルダの `roulette_settings.json` を削除して起動すると初期設定にリセットされます（ソースから実行している場合は `RRoulette\dist\roulette_settings.json`）。

**音が出ない**
→ 設定パネルの音量スライダーを確認してください。スピン音が「消音」になっている可能性があります。

**OBSに映らない**
→ OBSの「ウィンドウキャプチャ」で `RRoulette` を選択してください。「ゲームキャプチャ」では映りません。

**背景透過がOBSに反映されない**
→ OBSのウィンドウキャプチャソースのプロパティを開き、**「キャプチャ方法」を「Windows 10 (1903以降)」に変更**してください。
「BitBlt」など他のキャプチャ方法では背景透過がOBSに反映されません。
この手順は OBS Studio 32.1.0 (64 bit) で動作確認済みです。他のバージョンでは項目名・位置が異なる場合があります。

## ソースから実行する（開発者向け）

### 必要環境

- Windows 10 / 11
- Python 3.10 以上
- 依存パッケージ: `pygame`, `numpy`

### 依存パッケージのインストール

```
pip install -r requirements.txt
```

### 起動

```
python rroulette.py
```

### EXEのビルド

#### 開発用ビルド（通常）

`build_exe.bat` をダブルクリックするだけでビルドできます。

```
build_exe.bat
```

または手動でビルドする場合:

```
pyinstaller RRoulette.spec
```

ビルドが完了すると `dist\RRoulette.exe` が生成されます。

#### 公開用ビルド（任意署名あり）

GitHub Releases などに公開する場合は `build_release.bat` を使用してください。

```
build_release.bat
```

このスクリプトは以下を行います:

1. PyInstaller で EXE をビルド
2. `RROULETTE_SIGN_ENABLE=1` が設定されていれば、コード署名を試みる
3. 署名の成否をログに出力

**署名なしの場合:** 署名設定がなくてもビルドは成功します。未署名である旨のメッセージが表示されます。

**署名ありの場合:** `signing_env.example.bat` をコピーして実際の値を設定してから実行してください。

```
:: signing_env.example.bat をコピーして設定
signing_env.bat && build_release.bat
```

> **SmartScreen 警告と署名の関係**
>
> SmartScreen 警告は主に以下の要因で発生します:
> - コード署名証明書がない（発行元が「不明」と表示される）
> - 配布実績（reputation）が十分でない
>
> 署名なしで公開した EXE は、実行回数が少ない間は SmartScreen 警告が出やすい状態が続きます。
> 公開用コード署名証明書を導入することで、この警告を軽減・解消できます。
> 現時点では証明書未取得のため、配布 EXE は未署名です。

#### 将来の本格対応候補

以下の対応を将来検討しています:

- **公開用コード署名証明書の取得**: Sectigo / DigiCert 等の EV/OV 証明書
- **Microsoft Trusted Signing の利用**: Azure ベースの署名サービス（個人・OSS 向けにも対応）
- **リリース時のみ署名を行う運用**: CI/CD と組み合わせた自動署名フロー

現時点では上記のいずれも導入していませんが、`build_release.bat` と `signing_env.example.bat` を整備しており、証明書取得後すぐに署名フローを有効化できる状態にしています。

### 設定ファイルについて

`roulette_settings.json` はアプリと同じフォルダに自動生成されます。

- **EXE を使う場合**: EXE と同じフォルダに `roulette_settings.json` が作成されます
- **ソースから実行する場合**: スクリプトの隣の `dist\` サブフォルダに保存されます（`RRoulette\dist\roulette_settings.json`）

削除すると初期設定にリセットされます。

旧バージョンからの移行: `RRoulette\roulette_settings.json` が残っていても自動的に `dist\` へ移行されます（ウィンドウ位置を除く設定のみ引き継ぎ）。問題が残る場合は上記パスの `roulette_settings.json` を削除して起動してください。
